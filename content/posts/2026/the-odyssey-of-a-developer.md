# 万字长征：从 0 到 33 枚 Merged PRs，一个网络工程新生的三年全记录

## 01. 序言：邮政绿、代码绿与成长的频率

2026 年 7 月 8 日，柳州的夏雨初晴。

就在刚才，我拆开了那封中国邮政 EMS 录取通知书专递。**广西柳州城市职业学院，计算机网络工程专业。** 当我抚摸着通知书上烫金的文字时，我的另一只手下意识地刷新了 GitHub 主页。那个绿色的贡献矩阵（Contribution Graph）在屏幕上闪烁，334 个绿色的方格像是一串密电码，记录了我过去三年来每一个深夜的挣扎与突破。

如果说录取通知书是我现实世界的通行证，那么这三年的 GitHub 提交记录，就是我灵魂深处的编年史。

从 2023 年初出茅庐的“脚本小子”，到 2026 年拥有 33 枚高质量 Merged PRs 的开源贡献者，我走过了一段极度孤独却又异常灿烂的旅程。这篇文章，我想写给那个在旧笔记本前敲下第一行 HTML 的少年，也写给每一个正在代码世界里逆流而上的开发者。

---

## 02. 2023：启蒙时刻——那一抹“邮政绿”之前的“入门蓝”

三年前的今天，我的 GitHub 主页还是一片荒芜。那时候的我，对“开源”的理解仅仅停留在“去别人的仓库下载免费的代码”。

### 2.1 恩师引路：Web 世界的第一道光
幸运的是，在那个懵懂的起点，我遇到了一位带我真正“入行”的恩师。

那不是枯燥的课堂教学，而是一场场关于“互联网如何运行”的实战演习。他教我不只是如何写 HTML 标签，更多的是教我如何选购服务器配置、如何处理复杂的运维逻辑、如何让一个静态页面在 Linux 环境下优雅地跑起来。

在他的引导下，我学会了：
*   **不仅仅是 Coding**：我开始理解 CDN、反向代理（Nginx）以及域名解析的底层魅力。
*   **不仅仅是 Hosting**：我学会了如何权衡 CPU、内存与带宽的性价比，这种对硬件资源的敏感度，为我后来选择“网络工程”埋下了最深的伏笔。

### 2.2 硬件的“青铜时代”：那台 4 核 8G 的旧华硕
那时候，我的“武器库”还非常寒酸。高一那一整年，我守着一台好几年前的旧华硕笔记本。4 核 CPU、8G 内存，每当我打开 VS Code 和几个 Chrome 标签页，风扇的轰鸣声就像是在抗议。

但就在那样一台甚至运行稍微复杂的 Python 脚本都会卡顿的机器上，我完成了最初的 Git 启蒙。我学会了配置 SSH Key，学会了第一次用 `git push` 把代码推送到那个远在云端的仓库。那份成就感，是任何顶配硬件都无法替代的。

---

## 03. 2024：扩张之路——硬件迭代与“忙里偷闲”的助理生涯

进入 2024 年，我的 GitHub 贡献矩阵开始出现明显的深绿色块。这一年，我的生活节奏发生了剧烈的变化，技术栈也迎来了爆发。

### 3.1 武器升级：七彩虹将星 X15 的“火力全开”
高二那年，我终于迎来了我的“性能怪兽”——**七彩虹将星 X15 XS 22**。
原装的 i7-12700H 给了我处理大规模数据的底气。为了追求极致的多任务处理，我亲手将内存升级到了 **32GB**，并加装了一块 **2TB 的七彩虹“橘猫”硬盘**。

有了这 32G 内存，我终于可以肆无忌惮地开启数个 Docker 容器，运行虚拟机模拟网络环境。那块 2TB 的橘猫硬盘，则成了我所有开源项目和技术文档的“数字保险箱”。

### 3.2 校团委助理的“错位时空”
这一年，我身上多了一个特殊的身份：**校团委助理**。
在学校团委的全职助理工作中，我每天要处理海量的行政表格、组织活动、协调各个部门。在别人眼里，我似乎整天忙于琐事，离代码越来越远。

但只有我自己知道，我是在**“偷时间”**。
*   **忙里偷闲**：在两场会议的间隙，我会打开备忘录草拟一个函数逻辑。
*   **碎片化学习**：在值班的安静午后，我会掏出平板阅读 `aeon-toolkit` 的文档。
这种高度的时间管理能力，让我在繁忙的行政工作中，依然保持了 GitHub 贡献图的持续“点亮”。

## 04. 2025：深耕之年——在 33 枚 Merged PRs 里的成长与碰撞

2025 年是我技术思维彻底转型的一年。我开始不再满足于自己写小工具，而是尝试向全球顶尖的开源项目发起挑战。

### 4.1 只有“大型项目”才能带来的快感
很多人问我，为什么不去做一些小的、容易合并的 PR 来刷数据？
我的回答是：**平庸的合并毫无意义。**

我没有那种“小 PR 被合并也会开心”的平和心态。我是一个极致的结果导向者。对我而言，只有像 **aeon-toolkit** 这样具有国际影响力的、代码逻辑严密到近乎苛刻的大型项目，当我的代码被其维护者认可并 Merge 的那一刻，那种战胜自我的快感才足以覆盖所有的疲惫。

### 4.2 Sklearn 合约的捍卫战
在 [PR #3541](https://github.com/aeon-toolkit/aeon/pull/3541) 中，我修复了 `RandomChannelSelector` 的底层 Bug。
这个任务要求我深入理解 **Scikit-learn 的克隆合约 (Clone Contract)**。虽然在提交过程中没有遇到那种恶言相向的严厉 Review，但那种无形的、基于专业素养的博弈更加考验人。你需要用完美的测试用例、严密的逻辑推导去证明：**为什么你的方案才是 Right Fix**。

合并的那天，我坐在校团委办公室的电脑前，看到那个绿色的图标，我长舒了一口气。那是属于开发者的、最高规格的“战功记录”。

## 05. 网络基因：从路由器到小型 NAS

为什么在拥有了写出高性能前端架构和复杂 Python 逻辑的能力后，我依然选择了**计算机网络工程**作为大学专业？

### 5.1 家族传承的技术直觉
很多人到了高中才开始接触配置路由器，但我对计算机的接触几乎是从小开始的。我清楚地记得，小时候跟着家里的长辈一起研究如何配置拨号上网、如何调整路由器的信道、如何通过物理布线优化网络。

我从没有过“把家里网络搞瘫痪被父母吐槽”的尴尬经历，因为在我的家里，这种探索是被鼓励的。那种对网络拓扑的直觉，仿佛流淌在血脉里。

### 5.2 从家庭 NAS 到宿舍集群
在收到通知书之前，我已经在家中亲手组建了一套属于自己的 **小型 NAS 系统**。它承载了我的代码库、备份镜像以及我所有的数字化资产。
而在即将开启的大学生活中，我有一个非常“极客”的执念：**我一定要在宿舍里组建一套属于自己的 NAS 和微型私有云集群。** 

这不仅是为了存储，更是为了实践：我想在真实的局域网环境下，去复现我在代码中遇到的网络延迟、拥塞控制和安全防御。

## 06. 我的开发者准则：Tab、规范与强迫症

作为一个在 GitHub 摸爬滚打三年的开发者，我有一套近乎偏执的自我要求。

### 6.1 缩进的尊严：Tab 还是 Space？
虽然社区里关于空格与 Tab 的战争从未停止，但我是一个坚定的 **Tab 派**。在我看来，Tab 赋予了每个开发者根据自己喜好调整显示宽度的权利，这是一种代码层面的“兼容并包”。

### 6.2 规范即正义
我对代码美学的追求，不在于花哨的注释，而在于**极其规范的命名**与**严格遵循 Git 提交规范**。
*   `feat:`、`fix:`、`chore:`……每一个 Commit Message 都必须清晰地描述变动的动因。
*   变量命名必须见名知意，杜绝任何 `a`, `b`, `c` 这种毫无意义的字符。
这种对规范的坚持，让我在参与开源协作时，能迅速赢得维护者的信任。

## 08. 结语：长征刚刚开始，而我正值少年

录取通知书正静静地摆在我的显示器旁边。它的颜色和 EMS 包裹一样，是一种充满生机的、深邃的绿。

回望这三年，从那台 4 核 8G 的旧华硕，到现在的 32G 顶配将星；从连 HTML 标签都要查文档的萌新，到能在国际开源项目里捍卫 Sklearn 合约的贡献者。我最感谢的，是那个在校团委忙碌工作中、在备考压力下，依然坚持每天打开终端的自己。

**录取通知书不是终点，而是更高频道的起跑线。**
感谢那位教我运维与部署的恩师。
